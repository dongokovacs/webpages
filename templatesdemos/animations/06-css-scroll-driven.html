01 – view() fade-in
Elemek beúsznak amikor a viewportba érnek

Nincs JS listener. A böngésző figyeli az elem pozícióját és futtatja az animációt – animation-range: entry 0% entry 35%.

⚡

Nulla JS overhead

A böngésző natívan számolja az elem scroll pozícióját.

🔋

Energia-hatékony

Nincs requestAnimationFrame loop – csak CSS deklaráció.

🎯

Pontos timing

animation-range definiálja mikor indul és ér véget az animáció.

@keyframes fade-in-up {
  from { opacity: 0; transform: translateY(48px); }
  to   { opacity: 1; transform: translateY(0); }
}

.card {
  animation: fade-in-up linear both;
  animation-timeline: view();          /* scroll-hoz kötve */
  animation-range: entry 0% entry 35%; /* belép → 35%-ig tart */
}
02 – Scale animáció view()-vel
Elemek scale(0.1)-ről nőnek fel – görgetésre

Az elemek szinte láthatatlanok (0.1× méret, 0 opacity), és mikor a viewport-ba görgesz, felpattannak teljes méretükre. A spring easing (cubic-bezier(.34,1.56,.64,1)) rugalmasságot ad.

👆 Görgess le lassan – figyeld ahogy az elemek egyenként pattannak elő a semmiből. Ha nem látod az effektet, a böngésző nem támogatja a view() API-t.

/* "Pattanó" spring easing + scale(0.1) startból */
@keyframes scale-pop {
  from { opacity: 0; transform: scale(0.1) rotate(-8deg); }
  to   { opacity: 1; transform: scale(1)   rotate(0deg); }
}

.item {
  animation: scale-pop cubic-bezier(.34, 1.56, .64, 1) both;
  animation-timeline: view();
  animation-range: entry 5% entry 50%;
}

/* cubic-bezier(.34, 1.56, .64, 1) = spring/bounce easing
   – 1.56 az Y tengely, ez okozza a "túllövést" */
03 – scroll() progress
Scroll progress kör – animation-timeline: scroll()

Az SVG stroke-dashoffset animálva van a teljes oldal görgetéséhez kötve. Figyeld a kör kitöltését görgetés közben.

Oldal scroll progress

A kör az oldal tetejétől aljáig tölt be – animation-timeline: scroll() fogja a görgetést, nincs JS szükséges. Görgess egészen le az oldal aljáig a teljes kitöltéshez.

04 – CSS Parallax
Különböző rétegek eltérő sebességgel mozognak

Minden réteg animation-timeline: view()-t kap, de más-más translateY értékkel: lassú réteg ±80px, közepes ±140px, gyors ±220px. Görgess le és fel – figyeld a mélységi illúziót.

Közepes réteg ±140px

Lassú – háttér ±80px
Közepes – blob ±140px
Gyors – csillagok ±220px

/* Különböző rétegek = különböző translateY tartomány */

/* Lassú réteg – háttér */
@keyframes para-slow {
  from { transform: translateY(-80px); }
  to   { transform: translateY(80px); }
}
.layer-slow {
  animation: para-slow linear both;
  animation-timeline: view();
  animation-range: entry exit; /* teljes láthatóság alatt fut */
}

/* Gyors réteg – előtér */
@keyframes para-fast {
  from { transform: translateY(-220px); }
  to   { transform: translateY(220px); }
}
.layer-fast {
  animation: para-fast linear both;
  animation-timeline: view();
  animation-range: entry exit;
}

/* Minél nagyobb a translateY különbség, annál erősebb a parallax */
